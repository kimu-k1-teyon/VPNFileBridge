# skills/usage-create.md

## 目的
機能固有の実装詳細に依存しすぎない、汎用的な Usage ドキュメントを `Docs/Guides` 配下に作成する。

## 対象
- `Docs/Guides/**`
- 必要に応じて `overview.md` / `roadmap.md`
- 関連仕様（`Docs/Specifications/**`）

## 作成前の確認（最小）
- 対象機能名（例: upload / download / auth）
- 想定利用者（開発者 / 運用者 / QA）
- 実行環境（ローカル / ステージング / 本番）
- 必須入力値（環境変数、設定ファイル、認証情報）
- 成功判定（HTTPステータス、生成物、ログ）

## 手順
1) `overview.md` と `roadmap.md` を確認し、前提とスコープを合わせる  
2) 既存 `Docs/Guides` に重複・近似ドキュメントがないか確認する  
3) 出力先パスを決める  
   - 推奨: `Docs/Guides/<topic>-usage.md`（kebab-case）  
4) Usage 本文を作成する（下の必須セクションを満たす）  
5) コマンドはコピペ可能な形で記載し、例示値は「例」であることを明記する  
6) エラー対処を最低3件以上記載する（認証/入力/実行環境の3分類を推奨）

## 必須セクション
1. 目的  
2. 前提条件  
3. 事前準備  
4. 設定（環境変数・設定ファイル）  
5. 実行手順  
6. 結果確認（成功時の期待値）  
7. よくあるエラーと対処  
8. 更新ルール（値や仕様変更時にどこを直すか）

## ルール
- ドキュメントは手順中心にし、実装設計の詳細は `Docs/Specifications` へ分離する
- 環境依存パスは絶対パスと相対パスの両方を示す
- 例示の秘密情報はダミー値のみ使用する
- 成功条件は「確認コマンド + 期待出力」で定義する
- 読者が最短で再現できる順序（設定 → 実行 → 検証）を守る
